route(page, an, shouldReload)

The method is used to route between different controllers and hence, views(pages) in the application. The method will also automatically load all the images (using the loader class) specified in the controller's member "images".
Parameters:
Name Type Description
page string the name of the page to route to. The convention is to use the controller's class's name. For example "HomeController", the page will be "Home"
an object object to hold extra data the a page may need
shouldReload boolean the boolean indicates whether the application should refresh the page and unload all resources before entering the view(page). This is done to increase the performance of the overall application, as the application will slow down if page is not refreshed regularly.
